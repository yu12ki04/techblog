ZennもQiitaも使いたい。でも記事を二重管理するのはしんどい。

そんな問題を [C-Naoki/zenn-qiita-sync](https://github.com/C-Naoki/zenn-qiita-sync) が解決してくれます。Zenn形式で書いた記事をGitHub Actionsが自動でQiitaに変換・投稿してくれる仕組みです。

この記事ではその環境を一から作る手順をまとめます。

## 完成形

mainブランチにpushするだけで、ZennとQiitaの両方に記事が公開されます。

```
techblog/
├── .github/workflows/publish.yml  # 自動同期ワークフロー
├── articles/                      # Zenn記事をここに書く
├── books/
├── images/
└── qiita/public/                  # 自動生成されるQiita記事
```

## 手順

### 1. npm初期化 & CLI インストール

```bash
mkdir techblog && cd techblog
npm init -y
npm install zenn-cli @qiita/qiita-cli
```

### 2. Zenn・Qiitaを初期化

```bash
npx zenn init
npx qiita init
```

`qiita init` で `.github/workflows/publish.yml` が自動生成されますが、内容を差し替えます。

### 3. GitHub Actionsワークフローを書き換える

`qiita init` が生成するワークフローはQiitaへの投稿のみです。`C-Naoki/zenn-qiita-sync` に差し替えることで、ZennからQiitaへの自動変換も行われます。

```yaml:.github/workflows/publish.yml
name: Publish articles

on:
  push:
    branches:
      - main
      - master
  workflow_dispatch:

permissions:
  contents: write

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: false

jobs:
  publish_articles:
    runs-on: ubuntu-latest
    timeout-minutes: 5
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Run
        uses: C-Naoki/zenn-qiita-sync@main
        with:
          qiita-token: ${{ secrets.QIITA_TOKEN }}
```

### 4. qiita/public ディレクトリを作る

GitHubは空ディレクトリを管理できないため、`.gitkeep` を置きます。これがないとGitHub Actionsが `./qiita/public` を見つけられずエラーになります。

```bash
mkdir -p qiita/public && touch qiita/public/.gitkeep
```

### 5. GitHubリポジトリを作成してpush

```bash
git init
git add -A
git commit -m "Initial setup"
gh repo create techblog --public --source=. --remote=origin --push
```

:::message
GitHub CLIがない場合は `brew install gh` でインストールし、`gh auth login` で認証してください。
ワークフローファイルのpushには `workflow` スコープが必要です。
```bash
gh auth refresh --hostname github.com --scopes workflow
```
:::

### 6. シークレットを設定する

[Qiita > 設定 > アプリケーション](https://qiita.com/settings/applications) でアクセストークンを発行し（`read_qiita` + `write_qiita` スコープ）、GitHubリポジトリの `Settings > Secrets and variables > Actions` に `QIITA_TOKEN` として登録します。

### 7. ZennのGitHub連携

[Zenn Dashboard > GitHubからのデプロイ](https://zenn.dev/dashboard/deploys) でリポジトリを連携します。

## 記事の書き方

```bash
# 新しい記事を作成
npx zenn new:article

# プレビュー確認
npx zenn preview
```

記事が書けたらfrontmatterの `published: true` にしてpush。これだけでZennとQiitaの両方に公開されます。
